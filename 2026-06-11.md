# Отчет за 2026-06-11

## Задача

Настроить Wine для запуска PRO100 из папки Downloads.

## Отчет о выполнении

- Проверена система: NixOS, KDE Plasma, Wayland-сессия.
- Найден установщик: `/home/zephyr/Downloads/PRO100 5.45/PRO100 5.45 Setup.exe`.
- Подключен Wine 10.0 через Nix.
- Создан отдельный 32-битный Wine-префикс: `/home/zephyr/.wine-pro100`.
- Установлен PRO100 DEMO в префикс Wine.
- Создан запускатель: `/home/zephyr/.local/bin/pro100-demo`.
- Создан пункт меню: `/home/zephyr/.local/share/applications/pro100-demo.desktop`.
- Запускатель проверен в обычном оконном режиме Wine.
- В системную конфигурацию NixOS добавлен `wineWowPackages.stable`.
- Включена 32-битная графическая поддержка для Wine:
  - `hardware.graphics.enable = true;`
  - `hardware.graphics.enable32Bit = true;`
- Применена конфигурация через `nixos-rebuild switch`.

## Сложности и ошибки

- Изначально `wine` не был установлен в PATH.
- Первый запуск PRO100 показал ошибку графического адаптера.
- В логах Wine была ошибка 32-битного OpenGL: `couldn't initialize OpenGL`.
- Причина: в NixOS не была включена 32-битная Mesa/OpenGL-поддержка.
- После включения `hardware.graphics.enable32Bit` ошибка OpenGL исчезла.
- `sudo` не смог запросить пароль в неинтерактивном режиме, поэтому системные действия выполнялись через `pkexec`.
- Попытка отключить Wine `mshtml` и запускать через виртуальный рабочий стол `wine explorer /desktop=PRO100,1280x900` привела к нерабочему первому экрану выбора проекта.
- Рабочий вариант: обычный запуск через `wine DEMO.exe` с включенным `mshtml`.
